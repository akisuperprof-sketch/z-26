# 舌診アシスタント2025 仕様書

**バージョン:** 2.2
**最終更新日:** 2026-01-15

## 【① 前提・ゴール・非目標（Assumptions & Scope）】

-   **ゴール**: ユーザーがアップロードした舌の画像から所見の可能性を分析し、「所見→説明→推奨アクション」を自動生成する。健康管理への意識向上と、適切な受診行動のきっかけ作りを目的とする。
-   **想定ユーザー**: 健康意識の高い一般ユーザー。歯科・内科クリニックの待合室でのセルフチェック。企業の健康経営施策の一環。
-   **非目標**:
    -   医療的な**確定診断**、治療指示、投薬判断。
    -   本アプリは医療機器ではない。
-   **出力トーン**:
    -   断定的な表現を避け、「〜の可能性があります」「〜という所見が見られます」といった表現を用いる。
    -   緊急性の高い所見（レッドフラッグ）については、速やかな受診を明確に推奨する。

## 【② 主要機能一覧（Functional Requirements）】

| 機能群 | 機能 | 仕様要点 |
| :--- | :--- | :--- |
| **利用同意** | 免責事項の表示と同意 | アプリの目的と非目標（医療診断ではないこと）を明記し、ユーザーの同意を得てから利用を開始する。 |
| **ユーザー情報入力** | 基本情報の入力フォーム | 年齢、性別（必須）、身長、体重、最近気になること（任意）を入力する画面。 |
| **撮影サポート** | 撮影ウィザード＆ガイド | 4方向（正面, 左側縁, 右側縁, 舌裏）の画像アップロードを必須とする。撮影のポイントやイラストガイドを表示。 |
| **カメラ/アップロード** | **デバイス別撮影UI** | **Mobile**: `<input capture="user">`を使用し、ネイティブカメラを直接起動。<br>**PC**: `navigator.mediaDevices`を使用し、ブラウザ内でWebカメラを起動して撮影するカスタムUIを提供。<br>**選択モーダル**: 撮影スロットクリック時に「写真を撮る」「ライブラリから選ぶ」を選択可能。 |
| **開発者機能** | **カメラガイド (Dev)** | **開発者モード限定**: 顔認識に代わり、静的なオーバーレイガイド（口郭・舌郭・輝度判定）を表示する専用カメラUIを使用。 |
| **画像解析** | **AIによる画像解析** | アップロードされた画像とユーザー情報をGoogle Gemini APIに送信し、AIが所見をリアルタイムで分析する。解析中はアニメーション付きのローディング画面を表示。 |
| **画像解析 (拡張)** | **寒熱バランス判定 (Dev)** | **開発者モード限定**: 基準チャートと比較し、AIが「寒（-3）〜 熱（+4）」のスコアを算出。結果画面に専用メーターとチャートを表示する。 |
| **結果表示** | 解析結果の階層的表示 | **0. 寒熱バランス判定:** （Devモード有効時）スコアメーターとAI解説。<br>**1. 総合的な所見:** 健康面・体質面の特徴サマリー。<br>**2. 特に注意したい点:** 赤・黄リスクの所見詳細。<br>**3. 見られた所見:** 緑リスクの所見。<br>**4. 次のステップ:** 再撮影や専門家への相談推奨。 |
| **学習・リファレンス** | **舌診図鑑** | 主要な舌の所見（30項目以上）を一覧できる辞書機能。各所見には解説、リスクレベル、推奨アクションに加え、**医学的特徴を再現したAI生成イラスト**を表示（全項目対応済み ※現在は開発モード限定機能）。 |
| **インタラクティブ機能** | AIアシスタント | 各所見カード内で、より詳しい情報をAIに質問できる機能（Gemini APIを利用）。 |
| **再利用** | 再診断機能 | 結果画面から、再度撮影ウィザードに戻り、再診断を行える。 |

## 【③ 撮影要件・品質チェック】

-   **光源**: 逆光を避け、自然光に近い白色光の下での撮影を推奨。
-   **タイミング**: 食後や歯磨き後30分以上経過してから撮影することを推奨。
-   **角度・距離**: 4方向（正面・左側縁・右側縁・舌裏）の画像を必須とする。
-   **舌の出し方**: 力まずにリラックスして、軽く舌を出すようガイドする。

## 【④ 所見→説明文テンプレ（ルール判定の中心）】

※ アプリ内のルールデータベース（`constants.ts`）はフォールバックとして保持。主要な解析はAIが動的に生成する。

| 所見（AIが生成） | リスク色（AIが判断） | 推奨アクション（AIが生成） |
| :--- | :--- | :--- |
| イチゴ舌様 | 赤 | 速やかに受診（小児/内科） |
| 拭えない白斑 | 赤 | 2週間以上持続→歯科/口腔外科 |
| 青紫調（チアノーゼ） | 赤 | 直ちに救急受診 |
| 潰瘍/痛み（2週間以上） | 赤 | 歯科/口腔外科へ |
| 白苔びまん | 黄 | 舌清掃/水分/喫煙対策→経過観察 |
| 黒毛舌 | 黄 | 舌清掃・原因薬/喫煙の見直し |
| 平滑赤舌 | 黄 | 栄養/血液検査相談 |
| 歯痕（スカラップ） | 黄 | いびき/日中眠気→睡眠検査相談 |
| 巨舌傾向 | 黄 | 内科/耳鼻科で評価 |
| 地図舌 | 緑 | 経過観察/刺激物控えめ |
| 舌下静脈怒張 | 緑 | 経過観察/過度な圧迫回避 |

## 【⑤ 判定パイプライン（技術仕様）】

1.  **入力**: ユーザー情報と4方向の画像。
2.  **画像解析 (AI)**: ユーザー情報と4方向の画像データをGoogle Gemini APIに送信する。AIは提供された情報を基に舌の状態を分析し、所見のリストを構造化されたJSON形式で返す。
    -   *寒熱判定モード時*: 基準チャート画像を同時に送信し、比較スコアリングを実行する。
3.  **結果生成**: AIから返却された所見データ（および寒熱スコア）を処理・分類する。
4.  **NLG出力**: 分類された所見に基づき、`ResultsScreen.tsx` のロジックに従って「寒熱バランス（Dev）」「総合的な所見」「特に注意したい点」などの説明文を動的に生成して表示する。

## 【⑥ UI/UX フロー】

1.  **免責事項画面**: アプリの目的と注意点を確認し、同意チェックボックスを押下後、「同意して進む」ボタンで次へ。
2.  **基本情報入力画面**: 年齢・性別などの必須項目を入力し、「次へ進む」ボタンで撮影画面へ。
3.  **開発者モード分岐**: 設定アイコンまたは特定の操作により開発機能の設定が可能。
    -   **カメラガイド**: 静的オーバーレイガイドによる撮影補助。
    -   **診断モード選択**: 「通常モード」と「寒熱スコアリング（開発中）」の切り替え。
4.  **撮影ウィザード画面**: 
    -   スロットクリック時に「カメラ」「ファイル」を選択。
    -   PCの場合はブラウザ内WebカメラUI (`CameraCapture`) が起動。
    -   Mobileの場合も開発者モード時はWebカメラUI (`CameraGuideDev`) を強制起動可能。
5.  **解析中画面**: AIが解析中であることを示すアニメーションを表示。
6.  **結果画面**: 寒熱バランス（Dev）、サマリー、リスク別カード、ネクストアクションを表示。
7.  **舌診図鑑画面**: フッター等のリンクから遷移（または単独画面）。全所見リストを閲覧可能。

## 【⑦ データモデル】

-   **UserInfo**:
    -   `age`: `number` | `''`
    -   `gender`: `Gender` (Enum: '男性', '女性', 'その他') | `null`
    -   `height`: `number` | `''`
    -   `weight`: `number` | `''`
    -   `concerns`: `string`
-   **UploadedImage**:
    -   `slot`: `ImageSlot` (Enum: '正面', '左側縁', '右側縁', '舌裏')
    -   `file`: `File`
    -   `previewUrl`: `string`
-   **DiagnosisResult**:
    -   `heatCold?`: `HeatColdResult` (寒熱判定結果)
        -   `score`: `number` (-3 ~ +4)
        -   `label`: `string`
        -   `explanation`: `string`
    -   `findings`: `FindingResult[]`
-   **FindingResult** (extends Finding):
    -   `aiExplanation?`: `string` (AIによる個別解説)
    -   `key`: `string`
    -   `name`: `string`
    -   `riskLevel`: `RiskLevel`
    -   ...others

## 【⑧ 非機能要件（NFR）】

-   **パフォーマンス**: スムーズな画面遷移と、API応答に応じた適切なローディング表示。
-   **プライバシー**: 入力された情報や画像はサーバーに保存せず、セッション中のみメモリで保持する。APIキーは環境変数から読み込む。
-   **アクセシビリティ**: 操作性の良いUIコンポーネントと、十分なコントラスト比を確保。
-   **レスポンシブデザイン**: スマートフォンからデスクトップまで、様々な画面サイズで適切に表示される。

## 【⑨ 法務・安全・表現ガイドライン】

-   **医療表現**: 「診断」という言葉を避け、「解析」「所見の可能性」という表現に統一。
-   **免責事項**: アプリの初回起動時とフッターで、医療行為ではないことを繰り返し明記する。
-   **説明責任**: すべての所見に「なぜそのように考えられるか（理由）」と「次に取るべき行動（推奨アクション）」を併記する。
